## Affected pages/routes

- `/` — homepage (list-first per D6) — verify hero, search, near-me button, nearest cards render with geolocation grant/deny
- `/park/<slug>/` — park profile (all 47 live + 99 stubs as they're authored) — verify all 16 sections render gracefully with rich data and minimal data
- `/map/` — full Leaflet map page — verify 150+ markers cluster, popovers work, mobile pinch-zoom not locked
- `/builder/<slug>/` (×14) — must return HTTP 410
- `/shop/<slug>/` (×20) — must return HTTP 410
- `/api/suggestions` — POST endpoint with honeypot + Turnstile + Upstash rate-limit
- `/api/revalidate` — POST endpoint behind secret token for per-park ISR
- `/admin/lint` (dev-only) — surfaces malformed ParkLinks / RenovatedYears rows
- `/sitemap.xml` — every park URL, no builder/shop URLs
- `/robots.txt` — allow all

## Key interactions to verify

- Photo strip (D14): horizontal scroll on `/park/<slug>/`, tap → lightbox opens to all photos in order
- Status banner (D11): only renders for `Temporarily Closed` and `Permanently Closed`; Perm-Closed page stays live
- Programming module (D27): only renders when `Programming = true`; CTAs link to owner's coaching destinations
- ParkLinks renderer (D21): segments by Type — Connect section (Website/IG/FB/X/YT/TikTok) vs Support section (GoFundMe/Venmo/Patreon/Donate/GiveButter/PayPal) vs Other Links (unknown)
- Suggest an Edit modal (D28): opens, validates, hits `/api/suggestions`, shows success toast
- Geolocation "Find parks near me" (homepage): permission grant → list re-sorts; deny → city/zip input appears
- Get Directions (per park): opens native maps app on mobile (geo: URI), Google Maps on desktop

## Edge cases

- Park with no `park_photos` rows → no strip section rendered, no broken placeholder
- Park with `description = NULL` → "Overview" section hidden
- Park with no `park_builders` rows → "Built by" section hidden
- Park with no `park_links` rows → both Connect and Support hidden
- Park with `park_links.type = 'other'` → renders under "Other Links" section
- Park with `park_renovations` empty → no renovation pill on hero
- Park with multiple `park_renovations` rows → all rendered, sorted by `sort_order` then `year`
- Park with 0 nearby parks in 30mi → section hidden silently
- Park with 0 nearby shops in 30mi → section hidden silently
- Park within 30mi of an OH/NJ/NY shop → shop appears in Nearby Shops (no state border rule)
- Park with `park_amenities.present = false` AND notes/photo populated → row still rendered (notes/photo are meaningful even when amenity is absent — e.g., "no onsite shop, but Plank Eye is 0.3mi away")
- Suggestion submitted with honeypot field populated → returns 200 silently, no `suggestions` row
- Suggestion submitted without Turnstile token → 400
- 6th suggestion from same IP in 1hr → 429
- Suggestion submitted directly to Supabase using publishable key (`sb_publishable_*`, anon-role bound — bypass attempt) → RLS denies, returns 401/403
- Park slug with non-ASCII characters → URL encoding correct, sitemap entry correct
- `storage_path` references missing file in Supabase Storage → fallback to branded gradient hero (D13), log warning at build
- Webhook fires but `/api/revalidate` returns 5xx → Supabase retries automatically (E6); /admin/lint shows stale timestamp
- Webhook fires on `park_photos` row INSERT (not on parent `parks` row) → `/api/revalidate` resolves slug via JOIN, revalidates parent park (finding #3)
- Stub park with NULL `lat`/`lng` saved in Supabase Studio → profile page renders, excluded from /map/ and Nearby compute (finding #2)
- Migration script run twice for same park → child rows deleted-then-reinserted in transaction, no duplicates (finding #9)
- Suggestions API called with valid park_id but RLS WITH CHECK rejects (e.g., race with park deletion) → returns 500 with logged error (finding #15)
- Suggestion submitted from EU IP → submitter_ip stored as /24 CIDR, not full IP (finding #11)
- Supabase project pauses after 7 days idle → keepalive cron prevents (F1); if it fails, owner gets GitHub email
- Image asset request hits non-existent size → falls back to next-larger pre-rendered size (F2 strategy)

## Critical paths (must work end-to-end)

1. **Parent finds nearest park (P0):** Land on /, tap "Find parks near me", grant geo, see 5 nearest cards, tap one, land on profile, tap "Get Directions" → native maps opens with correct lat/lng.
2. **Parent reads a profile cold (P0):** Land on /park/fdr/ from Google search, all 16 sections render, hero photo loads in <2.5s on 4G simulation, no broken images.
3. **Closed park doesn't mislead (P0):** Permanently Closed park (e.g., one of the 2 in the audit) shows a prominent banner above the fold, page still readable as historical record.
4. **Suggest an Edit closes the loop (P1):** Open modal on /park/fdr/, type a correction, submit, see success toast, owner sees new row in the Postgres `suggestions` table via Supabase Studio.
5. **Owner authoring loop (P0):** Edit a record in Airtable, wait <10s, see the change reflected on /park/<slug>/ without a manual deploy.
6. **WP migration is replayable (P0):** Run the D4 migration script twice; second run produces zero duplicate records (idempotent via `wp_post_id UNIQUE` constraint).
7. **Retired URLs are gone (P1):** GET /builder/spohn-ranch-skateparks/ returns HTTP 410 with a body containing "permanently removed."
8. **SEO baseline (P0):** /sitemap.xml lists 150+ park URLs, no builder/shop URLs. Every /park/<slug>/ returns 200 with `<link rel="canonical">`.
