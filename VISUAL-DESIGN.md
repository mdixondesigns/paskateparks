## 1. Aesthetic direction — "After Hours at the Park"

A parent-warm but skate-authentic directory. Warm cream body keeps body content parent-readable on Android in sunlight; dark canvas moments (hero, Programming module, navigation, homepage entry) carry the alive/artistic energy. **Bright paint accents** (pink, yellow, teal, red, lime) sampled from the recycled-skateboard wood inspiration and the dark-with-paint reference give the system its color voice. **Recycled-skateboard pinstripe wood** is the system signature — used to mark the transitions between dark and light surfaces.

**Mood:** alive, hand-touched, fun, welcoming. Says "made by someone who knows skating and wants you to love it too." Not SaaS, not skate-niche, not civic-government, not editorial-polished.

**Memorable thing it serves** (the forcing answer from session start):
> *"Found something really unique and special. Very welcoming. Concerns addressed. Excited to explore."*

How the system serves it:
- **Pinstripe-wood motif** is the thing a parent will mention — distinctive in the category (no peer uses it), warm because it's craft-wood, instantly skate-recognizable.
- **Cream body + dark accent** is welcoming under sunlight (parent-on-Android-in-parking-lot is the P0 use case) and energetic at the moments that need to land.
- **Paint splashes on dark sections** carry the "alive / artistic" energy from the dark-with-bright-paint inspiration.
- **Hand-drawn skatepark obstacles** in the empty-stub state turn the directory's biggest content liability (99 of 150 parks are stubs at launch) into a brand-voice asset: "we're building this entry, here's where it'll be — help us fill it."

**Reference touchstones:**
- Recycled skateboard wood (Pool Product Design earrings, layered nightstand, lazy susan) — the literal source of the pinstripe palette and motif
- Dark walls with bright paint drips — the dark/paint inspiration the owner shared
- Cabinet Grotesk's wider, friendlier proportions vs. Inter/Geist

**Anti-references** (explicit not-this):
- AllTrails / NPS / Recreation.gov (too institutional and corporate-modern)
- Atlas Obscura editorial polish (too "adult")
- Concrete Disciples / Smap / skate-niche directories (skater-coded, alienating to parents)
- Generic SaaS dashboards (Inter + purple gradients + 3-column grids)

## 2. Typography

**Single family:** Cabinet Grotesk (Indian Type Foundry / Fontshare, **free**). Wide, friendly, modern grotesk with real character. Used at every level — display, headings, body, UI. No serif. The previous Curated Guidebook proposal used Fraunces (serif) — explicitly rejected by the owner for reading "too adult / too expensive."

**Weights used:**
- `900` (Black) — hero park names, top-of-page H1
- `800` (ExtraBold) — section headings, wordmark, nearby/archive card names
- `700` (Bold) — eyebrows, labels, amenity names, CTAs, navigation
- `500` (Medium) — secondary labels, captions, tags
- `400` (Regular) — body paragraphs, descriptions, hours stanzas

**Code / admin only:** JetBrains Mono (free, Google Fonts). Used on `/admin/lint` and inline `<code>` references only.

**Scale (mobile-first):**

| Token | Size | Weight | Use |
|---|---|---|---|
| `text-hero` | 56px | 900 | Park profile hero title |
| `text-home-hero` | 46px | 900 | Homepage H1 |
| `text-display` | 38px | 900 | Archive page H1 |
| `text-h2` | 36px | 800 | Doc-level H2 (`/about`, `/privacy`) |
| `text-h3` | 30px | 800 | Section headings inside profile body |
| `text-quote` | 28px | 800 | Programming module pull-quote |
| `text-card` | 18px | 800 | Nearby park name, archive card name |
| `text-body-lg` | 17px | 400 | Overview paragraph |
| `text-body` | 16px | 400 | Default body, row text |
| `text-body-sm` | 15px | 500 | Secondary body, amenity notes |
| `text-label` | 14px | 700 | Eyebrows, labels, chips, badges |
| **floor** | **14px** | — | **No font in production may go below 14px.** Owner's accessibility constraint. |

**Loading strategy:** Cabinet Grotesk from Fontshare via `<link rel="preconnect">` + Google-style stylesheet link. `font-display: swap`. Preload the 900 + 400 weights for hero rendering. JetBrains Mono lazy-loaded only on `/admin/*` routes (route-based code split).

**Letter-spacing:** `-0.025em` on display (900-weight hero), `-0.02em` on section headings, default (0) on body. Cabinet Grotesk reads tighter at scale; the negative tracking lets headlines breathe naturally.

**Line-height:** 0.93–1.05 on hero/display (tight, confident), 1.1–1.2 on section headings, 1.5–1.55 on body.

**Wood tones** (used only inside the pinstripe-wood motif, never as a foreground color):
- `--wood-tan` `#C9A87B`
- `--wood-pale` `#E5D5B3`
- `--wood-mid` `#A37A4D`
- `--wood-dark` `#6E4F2A`

**Semantic mapping:**
- success → `--lime`
- warning → `--yellow`
- error / danger → `--red`
- info → `--pink` (no separate info color — keeps palette tight)

**Contrast notes** (WCAG AA):
- `--text` on `--cream`: 17:1 — passes AAA
- `--cream` on `--ink`: ~14:1 — passes AAA
- `--cream` on `--pink`: ~4.2:1 — passes AA large/bold only; use bold weight ≥16px for pink-button labels
- `--ink` on `--yellow`: ~12:1 — passes AAA
- `--ink` on `--lime`: ~10:1 — passes AAA
- `--ink` on `--teal`: ~6.5:1 — passes AA
- `--cream` on `--red`: ~5:1 — passes AA
- **Bright accents on cream** (pink/yellow/teal/lime as fill colors against `--cream` page bg): use only as backgrounds with `--ink` or `--cream` foreground content, **never** as text on the cream page bg below 18px bold.

**Dark mode:** deferred to v1.1. The current system already has dark moments (hero, Programming, navigation) — a full inverted dark mode is the highest-cost addition for the value it provides given the parent-in-sunlight P0 use case. Document deferred plan; don't ship in v1.

## 4. The two divider systems (the system signature)

This is the visual logic that makes the whole system cohere. Two distinct dividers, two distinct jobs.

### 4a. The pinstripe — recycled-skateboard layered wood

**The thing every page has** in common. A horizontal band of vertical stripes in the brand palette + warm wood tones, mimicking layered colored skateboard plies pressed together. Sampled directly from the owner's recycled-skateboard inspiration (the lazy susan / nightstand / earrings).

**Implementation:** inline SVG data URI as a CSS `background-image`. Pattern repeats every 400px horizontally. ~95 colored vertical bars per repeat in variable widths (2–7px). Colors: pink, yellow, teal, red, lime, orange + wood-tan, wood-pale, wood-mid, wood-dark interwoven. CSS class: `.pinstripe`. Variants: `.pinstripe` (default 14px), `.pinstripe.thick` (22px), `.pinstripe.thin` (8px — **do not use**, owner rejected after v2).

**Where it appears (only at dark/light boundaries):**
- Bottom of `--ink` app header (every page)
- Bottom of `--ink` hero (populated park profiles)
- Top + bottom of `--ink` Programming module
- Bottom of `--ink` homepage hero (where dark meets cream list)
- Top of `--ink` cookie banner
- Top of `--ink` admin lint table
- Bottom of `--cream-warm` archive header (transition back to `--cream` list)
- Under each wordmark in the wordmark exploration cards (identity-defining variant)

**Where it MUST NOT appear:**
- ❌ Between cream-on-cream sections — that's the color rule's job
- ❌ On thumbnail bottoms (the 5px variant) — owner rejected this in v3
- ❌ Under the wordmark text in the live app header (the small underline was too thin)

**Rule of thumb per viewport:** if a parent's viewport shows the pinstripe more than 2 times, you're using it too much. It's a punctuation mark, not a paragraph.

### 4b. The color rule — pink / yellow / teal horizontal bands

**Section divider for cream-on-cream content.** Three horizontal bands stacked vertically, each ~2px tall, totaling 6px. Top band pink, middle yellow, bottom teal. Implemented as a CSS linear-gradient with hard color stops. CSS class: `.color-rule`.

**Where it appears:**
- Between every cream section inside park profile body (Address → Hours → Overview → Park Rules → Amenities → Connect → Support → Suggest → Nearby Parks)
- Between archive list cards (subtle — uses default `--hairline` not the color rule)
- Anywhere a section break is needed without crossing a dark/light boundary

**Rule:** color rule is decorative-functional, never identity-defining. The pinstripe is the brand; the color rule is connective tissue.

## 6. Layout

**Approach:** grid-disciplined body, dark/light contrast at hero + Programming + navigation moments.

**Mobile-first (P0 device):**
- Single column, full-bleed where appropriate (hero, programming, status banners)
- 22px horizontal gutters inside profile body
- Phone-width content area: 380px reference (iPhone 14 family width)

**Desktop max content widths:**
- Park profile: 720px (reads like a long article — preserves intimacy)
- Homepage / Map / Archive: 1200px
- Doc pages (/about, /privacy): 720px

**Border radius (hierarchical, not uniform):**

| Token | px | Use |
|---|---|---|
| `radius-xs` | 1 | color-rule edges |
| `radius-sm` | 4 | small chips, badges |
| `radius-md` | 6 | photo thumbs, swatches, internal tiles |
| `radius-lg` | 8 | cards, buttons, hero photo container |
| `radius-xl` | 12 | spec cards, programming module |
| `radius-2xl` | 16 | bottom sheet on map |
| `radius-3xl` | 32 | phone-frame (preview only) |
| `radius-full` | 9999 | EST. YYYY pill |

## 7. Motion

**Approach:** minimal-functional. Parents in parking lots don't want their phones to animate.

- **Page transitions:** 150ms fade
- **Image lightbox open:** 200ms zoom-from-origin
- **Focus rings:** instant (no transition)
- **Hover states:** 100ms color change on background-color
- **Reduced motion:** all transitions disabled when `prefers-reduced-motion: reduce` is set

**What we explicitly DO NOT do:**
- ❌ Scroll-driven animations
- ❌ Entrance animations on content
- ❌ Decorative loops (rotating logos, etc.)
- ❌ Parallax
- ❌ Skeleton-shimmer effects (use simple gray rectangles or fade-in instead)

## 8. Iconography

**Library:** Phosphor Icons (open-source, free, MIT). Three weight variants used:
- **Regular** — default for most icons (map pins, search, list, etc.)
- **Bold** — Park Rules vehicle row icons (more visual weight at 40px display size), amenity ✓ status, "Skateboards" / "Bikes" / etc.
- **Fill** — heart on Support button, map pins inside cluster markers

**Size standards:**
- 18px — inline body icons (row pins, arrow indicators)
- 20px — input icons (search, location)
- 22px — amenity status icons
- 28px–40px — Park Rules vehicle icons
- 36px — platform icons (Connect section)
- 56px+ — decorative, rare

**Color:**
- Default icon color in body: `--ink` or inherits text color
- Pin icon (address row, map sheet): `--pink`
- Amenity ✓ / Park Rules ✓: `--lime`
- Amenity minus / disallowed: `--text-muted`

**Known icon-name caveats (production cleanup):**
- For roller skates (Park Rules row): Phosphor has no dedicated roller-skate icon. v3 uses `ph-sneaker` as a stand-in; production should swap to a custom 24px SVG of a roller skate, OR keep `ph-sneaker` with the "Skates" label.
- `ph-scooter` exists but verify renders correctly.

## 9. Wordmark

**Primary mark:** "Pennsylvania" stacked over "Skateparks" (with "Skateparks" in the page's accent color — yellow on dark, pink on cream), set in Cabinet Grotesk 900, letter-spacing -0.025em. **Sitting on a slice of pinstripe-wood** as the visual identity foundation. No state-shape icon — the pinstripe carries the brand.

**Variants:**

| Context | Treatment |
|---|---|
| **Primary (light backgrounds)** | "Pennsylvania" in `--text`, "Skateparks" in `--pink`, pinstripe underline 8px tall × 160px wide |
| **Dark (app header, footer)** | "Pennsylvania" in `--on-ink`, "Skateparks" in `--yellow`, full-width pinstripe at the bottom of the dark surface |
| **Compact (favicon, OG image)** | "PA Skateparks" inline, pinstripe underline 120px wide |
| **Inline (app header live)** | Inline `Pennsylvania Skateparks` on one line, "Skateparks" accent-colored, no underline pinstripe — the pinstripe at the bottom of the dark app header serves the same purpose |

**Rule:** the wordmark always sits on or near a pinstripe. The pinstripe IS the visual identity.

## 10. Photography treatment

**Hero photos:**
- Full-bleed at top of populated park profiles
- Aspect ratio approximately 375 × 420 mobile (~9:10) — gives room for hero title + EST pill + renovated tag
- Light desaturation (~10–15%) applied via Sharp at build time
- Subtle film grain (4–8% noise) applied via Sharp at build time
- 30% bottom gradient overlay (`rgba(15,11,8, 0.0)` → `rgba(15,11,8, 0.85)`) so hero text holds against any photo
- Bottom 14px replaced by the pinstripe motif (the dark/light transition)

**Photo strip (Overview section):**
- 96–110px square thumbnails
- 10px gap
- Horizontally scrollable strip, scroll snap to each thumb
- Tap → opens lightbox starting at that index (D14 / D29)

**Amenity photos:**
- 84–96px wide × 64–72px tall
- 6px border-radius
- Optional per amenity row
- Photo + italic notes shown below the row's name when populated

**Thumbnails (Nearby park / Home park / Archive card):**
- 64–80px square
- 8px border-radius
- No pinstripe underlay (removed in v3)
- For stub-park cards: use the cream-warm + amber + pink gradient (matches stub hero treatment) instead of a photo placeholder

**Sharp build-step requirements:**
- 3 sizes per photo: 400w / 800w / 1200w
- WebP format (`<picture><source srcset>` with JPEG fallback)
- All sizes pre-rendered at migration via Sharp, uploaded to Supabase Storage (per STACK-PIVOT.md F2)

## 11. Empty-stub state (the 99-of-150 critical path)

99 of 150 parks will launch with minimal data per [DESIGN.md](DESIGN.md) and [SITE-AUDIT.md](docs/archive/SITE-AUDIT.md). The stub state is a P0 visual responsibility.

**Hero treatment:**
- Background: `--cream-warm` with soft radial gradients of `--yellow` (top-right, ~35% opacity) and `--pink` (bottom-left, ~22% opacity)
- Hand-drawn skatepark obstacles SVG sitting behind the title at ~42% opacity: quarter pipe (curved arc + coping line) + flat rail (with two angled supports) + manny pad (rounded rectangle) + dashed ground line. Stroke color `--text`, stroke-width 2.4, stroke-linecap and linejoin round. 240×140 viewBox.
- Eyebrow: city / state in `--pink` ALL-CAPS 14px 700 tracked 0.14em
- Title: park name in Cabinet Grotesk 900 56px `--text`
- Stub-meta line: *"We're still building this guidebook entry. Have a photo, an opening-hours correction, or just know the park well? Help us fill it in."*
- CTA: yellow filled "Suggest an edit →" pill (uses `--yellow` background, `--ink` text, 12×16 padding, 8px radius)

**Below hero:**
- Pinstripe transition (dark/light boundary in spirit, even though this hero is light — the pinstripe still marks the transition from "hero" to "details")
- Address row only (if known, often just "Ambler, PA")
- Color-rule divider
- Section heading: *"Still gathering details"*
- One-line body: *"When this park is filled in, you'll see hours, photos, amenities, the surface type, and who built it — same as every other park in the guidebook."*
- Full-width `--ink` button: "See nearby parks on the map" (so the parent isn't dead-ended)

**Why this works:** the stub is presented as a *guidebook page still being written*, not a broken or empty page. The obstacle drawing signals "skatepark goes here." The voice (*"Have a photo? Help us fill it in"*) turns a content gap into a contribution invitation.

## 12. Programming module — the conversion moment (D27)

When `parks.programming = true`, render this module after the Amenities section in the profile body. CTAs are template-level, not per-park.

**Composition:**
- Full-width `--ink` band (extends to phone edges via negative margin)
- Pinstripe at top AND bottom of the band (the dark/light frame)
- Two paint splashes positioned absolutely as background:
  - Hot pink blob top-right, ~110px (`opacity: 0.7`)
  - Yellow blob bottom-left, ~80px (`opacity: 0.5`)
- Eyebrow: *"For parents · the coaching team"* in Cabinet Grotesk 700 14px tracked 0.14em, `--yellow` color
- Pull-quote heading: Cabinet Grotesk 800 28px line-height 1.1, with at least one key word in `--pink`. Example: *"New to skateparks? You're **not alone** — most parents aren't either."*
- Body: 15px Regular, `--on-ink-muted` color, max 56ch
- CTA stack (vertical, full-width):
  - **Primary:** `--pink` filled, `--on-ink` text — "Find a coach in [park city]" + right arrow icon
  - **Secondary:** transparent with `--on-ink-muted` border — "Become a coach"

**Why this works:** the module reads as an editorial pull-quote (italic-coded heading + paint splashes + intimate copy), not a sales banner. Highest-conversion treatment without ever feeling like advertising — funnels parents to the coaching business via curatorial voice.

## 13. Status banners (D11)

Per D11, status is silent when Open, banner-prominent when Closed. Page stays live for Permanently Closed (historical record).

**Temporarily Closed:**
- Full-width `--yellow` band at top of profile (above hero)
- 16px 22px padding
- Label line: *"NOTE FROM THE EDITOR"* in Cabinet Grotesk 800 14px ALL-CAPS tracked 0.1em, in `--ink`
- Body: 15px 500, `--ink` color, ~1.5 line-height
- Example body: *"Bayne Skatepark is closed for resurfacing through April 2026. Locals tell us it'll be worth the wait."*

**Permanently Closed:**
- Same composition but `--red` background, `--cream` foreground
- Label line: *"THIS PARK IS NO LONGER OPEN"*
- Body: *"We're keeping this page as a historical record of what was here."*
- Park name renders as normal but with date range in city/EST line: *"Love Park (historical) · Philadelphia, PA · 1989 – 2016"*

## 14. Connect / Support (D21, D23)

### Connect section (Website, Instagram, Facebook, Twitter, YouTube, TikTok)

Each link is a horizontal row in a `--surface`-white container with `--hairline-strong` border, 8px radius, 12×14 padding. Layout:

```
[36px icon tile] [Platform label (700, 16px)]      [arrow ↗]
```

The icon tile carries **platform color identity** (cohesive frame, distinct colors):

| Platform | Tile background | Tile icon color |
|---|---|---|
| Website | `--ink` | `--cream` |
| Instagram | `--pink` | `--cream` |
| Facebook | `--teal` | `--ink` |
| Twitter | `--ink` | `--cream` |
| YouTube | `--red` | `--cream` |
| TikTok | `--ink` | `--cream` |

This restores the at-a-glance platform scanning that v1's all-forest treatment lost, without going back to the mockup's clashy-stacked-buttons problem.

### Support section (GoFundMe, Venmo, Patreon, Donate, GiveButter, PayPal)

Single `--red` filled button with heart icon (Phosphor fill). Same 8px radius, 16×20 padding, 16px 700 label. The brick red signals "support / heart" without alarm.

## 17. Accessibility

- **Min font size: 14px.** Owner accessibility constraint. No exceptions in production.
- **Contrast: WCAG AA minimum**, AAA where possible (most body text exceeds AAA naturally on cream/ink).
- **Tap targets: 44pt minimum** for all interactive elements (Apple HIG / WCAG 2.5.5).
- **Focus rings: visible, high-contrast.** 2px solid `--pink` outline with 2px offset for buttons/links. Instant (no transition delay).
- **Reduced motion:** all animations / transitions disabled when `prefers-reduced-motion: reduce`. Implemented via media-query in global CSS.
- **Alt text:** all park photos must have alt text. Owner backfills per TODOS.md P1. Auto-generated fallback: `"<Park name> photo <index>"` per D29.
- **Color is never the only signal.** Vehicle disallowed = icon faded + diagonal slash + line-through label (three signals, not just the red slash).
- **Keyboard navigation:** all interactive elements reachable via Tab; modal traps focus; Escape closes modals; arrow keys navigate photo lightbox.

## 18. Loading / Performance

Per the project performance budget (TTI < 3s, LCP < 2.5s on 4G Android):

- **Typography:**
  - Cabinet Grotesk via `<link rel="preconnect">` + Fontshare stylesheet
  - `font-display: swap`
  - Preload weights 900 and 400 in `<head>`
  - JetBrains Mono lazy-loaded only on `/admin/*`
- **Icons:**
  - Phosphor regular + bold + fill loaded via npm package (tree-shake on import)
  - Or via the Phosphor web CSS only when using `<i class="ph">` markup (vendored to /public/phosphor.css to avoid CDN dependency)
- **Pinstripe motif:**
  - Pure CSS (`background-image: url("data:image/svg+xml;utf8,..."")`), no external asset
  - Cacheable forever in the stylesheet
- **Paint splashes:**
  - Inline SVG in markup (not external assets)
  - Reused across pages via template
- **Hero photos:**
  - `<picture><source srcset>` with 400w/800w/1200w WebP
  - `loading="eager"` + `fetchpriority="high"` on the populated park profile hero
  - All other photos `loading="lazy"`
- **CSS:**
  - Single critical-CSS extraction for above-the-fold rendering
  - Tailwind tree-shaken to production-only classes

## 19. Known TODOs (production cleanup)

- **Roller-skate icon:** v3 uses `ph-sneaker` as a stand-in in the Park Rules row. Production should ship a custom 24px SVG of a roller skate (free, hand-drawn) OR keep `ph-sneaker` with the label "Skates" — owner decides.
- **Hero photo treatment:** the v3 preview uses CSS gradient placeholders. Production photos need the Sharp build step: ~10–15% desaturation + 4–8% film grain + the bottom 30% gradient overlay baked in.
- **Stub-state generative gradient:** v3 uses a single fixed gradient. Production should hash the park slug to vary the gradient subtly per-park (so two adjacent stubs in the list don't look identical).
- **Dark mode:** deferred to v1.1. When revisited, the existing `--ink` / `--on-ink` tokens are 80% of the work.
- **Map page marker design:** v3 uses a CSS-built pin shape. Production may need a real SVG that matches Leaflet's positioning expectations more cleanly.
- **Wordmark mark for favicon:** the wordmark relies on the pinstripe. For a 32×32 favicon, the pinstripe is too small to read — either use a compressed 3-band-of-color version, or design a dedicated favicon mark.
